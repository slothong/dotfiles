아래 규칙을 준수함으로써 유저 경험과 동일한 테스트를 지향한다.
- API 모킹은 MSW를 사용한다.
- 유저 이벤트는 `@testing-library/user-event`를 사용한다.
- 쿼리 우선 순위: `getByRole`, `getByLabelText`, `getByText` 등 접근성 기반 쿼리를 우선 사용한다. `getByTestId`는 최후의 수단으로 사용한다.
- 컴포넌트 내부 상태나 메서드를 직접 테스트하지 않는다. 화면에 보이는 결과만 검증한다.
- 비동기로 나타나는 요소는 `findBy*` 쿼리를 사용한다. 수동으로 `waitFor`로 감싸지 않는다.

아래 규칙을 준수함으로써 테스트 코드의 신뢰성과 유지보수성을 확보해야 한다.
- 테스트는 "구현"보다 "요구사항"을 검증해야 한다.
- 하나의 테스트는 하나의 의도를 말해야 한다.
- 불필요하게 깨지기 쉬운 테스트는 피해야 한다.

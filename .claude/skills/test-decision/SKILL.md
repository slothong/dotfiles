# 테스트 작성 판단

구현할 코드(또는 구현된 코드)를 분석하여 테스트 코드 작성이 필요한지 판단하고, 근거와 함께 결론을 제시합니다.

## 작동 방식

1. 대상 코드를 읽거나 설명을 분석한다
2. 아래 판단 기준에 따라 평가한다
3. **결론**, **근거**, 테스트가 필요한 경우 **테스트 전략 개요**를 출력한다

---

## 입력

$ARGUMENTS

인자가 파일 경로면 해당 파일을 읽는다. 인자가 없으면 사용자에게 대상 코드 또는 구현 설명을 요청한다.

---

## 판단 기준

아래 기준을 **순서대로** 적용한다. 해당하는 항목이 많을수록 테스트 필요성이 높다.

### 테스트가 필요한 코드 (높은 가치)

| 기준 | 설명 |
|------|------|
| 비즈니스 로직 | 도메인 규칙, 계산, 조건 분기가 포함된 로직 |
| 데이터 변환 | 입력을 가공하여 다른 형태로 출력하는 함수 (매퍼, 포매터, 파서 등) |
| 조건부 렌더링 | 상태나 props에 따라 다르게 표시되는 UI 컴포넌트 |
| 에러 처리 | API 실패, 예외 상황에 대한 분기 처리 |
| 사용자 인터랙션 | 클릭, 입력, 폼 제출 등 사용자 행동에 반응하는 로직 |
| 복잡한 상태 관리 | 여러 상태 간 전환, 파생 상태, 비동기 상태 조합 |
| 유틸리티 함수 | 여러 곳에서 재사용되는 순수 함수 |
| API 통합 | API 호출과 응답 처리가 포함된 코드 |

### 테스트가 불필요한 코드 (낮은 가치)

| 기준 | 설명 |
|------|------|
| 타입 정의 | interface, type, enum 만 포함된 파일 |
| 상수/설정 | 단순 값 나열 (config, constants, theme 등) |
| 단순 재수출 | `export { X } from './x'` 같은 배럴 파일 |
| 프레임워크 설정 | Next.js layout, provider 래퍼 등 프레임워크가 요구하는 보일러플레이트 |
| 단순 표시 컴포넌트 | props를 그대로 렌더링만 하는 컴포넌트 (로직·분기 없음) |
| 생성된 코드 | 코드 생성기가 만든 파일 (API 클라이언트 등) |
| 외부 라이브러리 래퍼 | 라이브러리 함수를 단순 위임하는 얇은 래퍼 |

## 판단 프로세스

### 1단계: 코드 분류

대상 코드를 읽고 다음을 파악한다:

- **파일 유형**: 컴포넌트 / 훅 / 유틸리티 / API / 타입 / 설정
- **복잡도**: 분기(if/switch/삼항)의 수, 상태 변수의 수
- **의존성**: 외부 API 호출, 다른 모듈과의 결합도
- **재사용성**: 이 코드를 사용하는 곳이 여러 곳인가

### 2단계: 점수 산출

아래 항목에 해당하면 각각 +1점을 부여한다:

- [ ] 조건 분기가 2개 이상 존재
- [ ] 비즈니스 규칙이나 도메인 로직이 포함됨
- [ ] 데이터를 변환하거나 가공하는 로직이 있음
- [ ] 에러/예외 처리 경로가 존재
- [ ] 사용자 인터랙션을 처리함
- [ ] API를 호출하고 응답을 처리함
- [ ] 여러 곳에서 재사용됨
- [ ] 복잡한 상태 전환이 있음

**점수 해석**:
- **0점**: 테스트 불필요
- **1-2점**: 테스트 선택적 (변경 빈도가 높으면 권장)
- **3점 이상**: 테스트 필요

### 3단계: 최종 판단 보정

점수와 무관하게 아래 조건을 확인한다:

- 타입 정의만 있는 파일 → **무조건 불필요**
- 상수/설정 파일 → **무조건 불필요**
- 배럴 파일 (index.ts 재수출) → **무조건 불필요**
- 버그 수정 코드 → 점수와 무관하게 **회귀 테스트 필요**

## 출력 형식

다음 형식으로 결과를 출력한다:

```
## 테스트 판단 결과

**대상**: [파일 경로 또는 설명]
**결론**: ✅ 테스트 필요 / ⚠️ 테스트 선택적 / ❌ 테스트 불필요
**점수**: N/8

### 판단 근거

- [해당하는 기준과 그 이유를 구체적으로 나열]

### 테스트 전략 (테스트가 필요한 경우)

- **테스트 유형**: 단위 테스트 / 통합 테스트 / 컴포넌트 테스트
- **핵심 테스트 케이스**:
  1. [테스트 케이스]
  2. [테스트 케이스]
  ...
- **모킹 대상**: [MSW로 모킹할 API 등]
```
